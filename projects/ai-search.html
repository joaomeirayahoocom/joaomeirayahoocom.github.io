What is RAG?

Retrieval-Augmented Generation solves a fundamental LLM limitation: language models only know their training data. RAG augments the LLM's knowledge by retrieving relevant content from YOUR documents at query time.

Retrieval

Find relevant documents using vector search and semantic matching

Augmented

Inject retrieved document chunks into LLM context

Generation

GPT-4 generates natural answers using YOUR documents

Zero Secrets Architecture

# OIDC - No client secrets
provider "azurerm" {
  use_oidc = true
}

# Storage without keys (CAF compliant)
resource "azurerm_storage_account" "documents" {
  shared_access_key_enabled = false
}

# Service-to-service via managed identity
resource "azurerm_role_assignment" "search_to_openai" {
  role_definition_name = "Cognitive Services OpenAI User"
  principal_id         = azurerm_search_service.main.identity[0].principal_id
}
